# Incident Response

Detecting, containing, investigating, and recovering from security incidents through structured processes and forensic analysis.

---

## Tools & Repositories

- [velociraptor](https://github.com/Velocidex/velociraptor) — Live endpoint forensics and response platform
- [volatility3](https://github.com/volatilityfoundation/volatility3) — Memory forensics framework
- [autopsy](https://github.com/sleuthkit/autopsy) — Digital forensics platform
- [plaso](https://github.com/log2timeline/plaso) — Timeline forensics (log2timeline)
- [KapeFiles](https://github.com/EricZimmerman/KapeFiles) — KAPE artifact collection targets
- [TheHive](https://github.com/TheHive-Project/TheHive) — Incident response case management platform
- [Cortex](https://github.com/TheHive-Project/Cortex) — Observable analysis and active response engine
- [rita](https://github.com/activecm/rita) — Detect C2 beaconing through network analysis
- [Aurora-Incident-Response](https://github.com/cyb3rfox/Aurora-Incident-Response) — IR documentation framework
- [OSSEM](https://github.com/OTRF/OSSEM) — Open Source Security Events Metadata

## Books & Learning

| Book | Author | Why Read It |
|---|---|---|
| Incident Response & Computer Forensics | Luttgens et al. | Structured IR methodology — what to do when the alarm fires |
| Blue Team Field Manual (BTFM) | Clark & Robertson | The defender's desk reference for IR tasks |
| Applied Incident Response | Steve Anson | Practical IR with Windows forensics and threat intel context |
